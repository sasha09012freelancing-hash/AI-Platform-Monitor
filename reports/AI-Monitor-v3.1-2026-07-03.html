3. Детали по платформам

Vertex AI: Google выпустил ADK (Agent Development Kit) для построения масштабируемых multi-agent систем, решающий проблему "context collapse" в enterprise-окружениях (DP). Это усиливает критерий Prod-ops, так как ADK предоставляет инструменты для управления сложными workflows и динамическими бизнес-правилами (CP). Для SL2 AI Platform это давление на G3 (нет inter-agent tracing) и G7 (зрелость), так как Google предлагает готовое решение для enterprise-масштабирования, в то время как SL2 AI Platform требует ручной настройки observability. Однако ADK остаётся частью managed-сервиса, что сохраняет преимущество SL2 AI Platform в self-hosted сегменте.

Сигналы: Beyond Static Prompts with Google ADK, Claude Apps Gateway

AWS Bedrock: AWS вывел в GA AgentCore Runtime с изоляцией каждого агента в отдельной microVM, что закрывает критерии Indep. deploy, Indep. scale и Secrets/tenant (DP). Это прямое усиление позиции Bedrock в enterprise-сегменте, так как теперь агенты могут масштабироваться независимо с гарантированной изоляцией (CP). Для SL2 AI Platform это угроза по G3 (observability) и G4 (cost tracking), так как Bedrock предлагает managed-решение с готовыми инструментами для мониторинга и финансового контроля, в то время как SL2 AI Platform требует интеграции сторонних инструментов (например, OpenTelemetry). Также Bedrock представил serverless A2A gateway, что частично закрывает G2 (lock-in в Agent Protocol), но пока стандарт A2A не устоялся.

Сигналы: AgentCore Runtime GA, Structured Memory Filtering

AutoGen: В обсуждениях на GitHub появились предложения по security-инфраструктуре для агентов, включая библиотеку `proveyouragent` для аутентификации агентов с использованием Ed25519 и `AgentGraph` для trust verification (DP). Это усиливает критерий Secrets/tenant, так как предлагает стандартизированный подход к управлению идентичностью и доступом в multi-agent системах (CP). Для SL2 AI Platform это давление на G9 (молодой проект без community) и G3 (observability), так как AutoGen предлагает open-source решения для проблем, которые SL2 AI Platform решает вручную. Однако AutoGen не предоставляет полноценной платформы с self-host и multi-tenancy, что сохраняет нишу для SL2 AI Platform.

Сигналы: Agent Authentication Discussion, AgentGraph Trust Verification

AgentForce: Salesforce представил Mobile Skills для быстрого создания native-приложений и опубликовал гайд по MCP Server Security, включая аутентификацию, авторизацию и логирование (DP). Это усиливает критерий Code+No-code, так как Mobile Skills позволяют разработчикам генерировать приложения с помощью простых промптов, а также Prod-ops за счёт улучшенной безопасности (CP). Для SL2 AI Platform это давление на G1 (нет visual builder) и G6 (LLM gateway), так как Salesforce предлагает более зрелые инструменты для no-code разработки и security. Однако AgentForce остаётся частью экосистемы Salesforce, что ограничивает его применение в self-hosted окружениях.

Сигналы: Mobile Skills GA, MCP Server Security

n8n: Вышел гайд по MCP Server Security и статья о agentic AI design patterns, включая governance и error recovery (DP). Это усиливает критерий Prod-ops, так как n8n предлагает практические рекомендации по безопасности и надёжности multi-agent систем (CP). Для SL2 AI Platform это давление на G3 (observability) и G5 (data-residency), так как n8n активно развивает инструменты для enterprise-безопасности, в то время как SL2 AI Platform полагается на внешние SaaS-решения (например, LangSmith). Однако n8n не предоставляет полноценной изоляции агентов (Indep. deploy/scale), что сохраняет преимущество SL2 AI Platform.

Сигналы: MCP Server Security, Agentic AI Design Patterns

NetBox: Вышел HIPAA Compliance Pack для проверки сетевой сегментации и инвентаризации активов, а также Asset Lifecycle GA для управления закупками (DP). Это усиливает позицию NetBox в enterprise-сегменте, особенно для регулируемых отраслей, но не затрагивает напрямую критерии SL2 AI Platform (CP). Для SL2 AI Platform это косвенное давление на G7 (зрелость), так как NetBox демонстрирует зрелость в смежной нише (IT asset management), но не является прямым конкурентом.

Сигналы: HIPAA Compliance Pack, Asset Lifecycle GA

Snipe-IT: В обсуждениях на GitHub предложена модернизация UI/UX дашборда (DP). Это не затрагивает напрямую критерии SL2 AI Platform, но может усилить привлекательность Snipe-IT для пользователей, ищущих простые решения для asset management (CP). Для SL2 AI Platform это не является прямой угрозой, но подчёркивает важность G1 (нет visual builder) для пользователей без технического бэкграунда.

Сигналы: Modernize Dashboard UI
