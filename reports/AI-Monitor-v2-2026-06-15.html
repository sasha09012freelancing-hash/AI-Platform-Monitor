3. Детали по платформам

Vertex AI:

Google активно развивает инфраструктуру для агентных систем, уделяя внимание конфиденциальности и интеграции данных. Основной упор сделан на работу с Foundation Models в контексте безопасности данных и бизнес-аналитики.

Сигналы: развитие Confidential AI, интеграция агентов в Looker

AWS Bedrock:

AWS агрессивно внедряет протокол MCP для унификации взаимодействия агентов с внешними данными и инструментами. Платформа делает ставку на автоматизацию процессов через Bedrock Data Automation и улучшение оценки качества работы агентов.

Сигналы: оптимизация операций с помощью агентов, внедрение Agent-EvalKit

Azure Foundry:

На данной неделе существенных обновлений или публичных анонсов по платформе зафиксировано не было.

Сигналы: страница обновлений Azure

AutoGen:

Сообщество смещает фокус на Enterprise-требования: безопасность памяти, аудит-трейлы и защиту от вредоносных действий агентов. Разрабатываются механизмы предотвращения зацикливания и неконтролируемых затрат API.

Сигналы: внедрение Agent Memory Guard, реализация Governance-хуков

AgentForce:

Salesforce продолжает глубокую интеграцию своей экосистемы с AI-агентами через новые SDK и навыки (Skills). Основное внимание уделено созданию production-ready решений внутри существующей CRM-архитектуры.

Сигналы: релиз Summer '26, разработка приложений через Salesforce Skills

LangGraph:

Продолжается работа над стабильностью CLI и повышением удобства разработки агентных графов. Фокус остается на поддержке широкого спектра LLM-провайдеров через консистентные библиотеки.

Сигналы: обновление langgraph до 1.2.5, улучшение langgraph-cli

n8n:

Платформа укрепляет позиции в Enterprise-сегменте, интегрируя OpenTelemetry для глубокой наблюдаемости. Активно продвигаются лучшие практики по LLM-роутингу и управлению сложными агентными пайплайнами.

Сигналы: поддержка OpenTelemetry, гайды по LLM-роутингу

5. Выводы для команды

  1. MCP как стандарт: AWS делает ставку на MCP (Model Context Protocol). SL2 AI Platform должна обеспечить нативную поддержку или полную совместимость с этим протоколом, чтобы избежать изоляции экосистемы.
  2. Governance и безопасность: Конкуренты (AutoGen, n8n) активно закрывают брешь в Enterprise-governance (аудит, трейсинг, защита памяти). Нам необходимо убедиться, что SL2 AI Platform имеет превосходство в нативной изоляции данных tenants.
  3. Наблюдаемость: n8n показывает, что клиенты требуют интеграции с существующими стеками (OpenTelemetry). Рассмотреть возможность стандартизации телеметрии в SL2 AI Platform для соответствия корпоративным стандартам логирования.
  4. Конкуренция в Enterprise: Salesforce (AgentForce) и Google (Vertex) уходят в вертикальные решения (CRM, Data Analytics). SL2 AI Platform выигрывает за счет self-hosted гибкости, поэтому фокус на "Local-first / Self-hosted" должен стать ключевым маркетинговым преимуществом в ответах на Enterprise-запросы.
