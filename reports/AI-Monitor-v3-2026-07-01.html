3. Детали по платформам

Vertex AI: Google расширил Gemini Enterprise Agent Platform, добавив fully-managed remote MCP-серверы и conversational analytics в BigQuery (DP). Это усиливает критерий Prod-ops за счёт интеграции с Cloud Monitoring и anomaly detection (CP). Для SL2 AI Platform это давление на GAP G5 (self-hosted observability) и G4 (cost tracking), так как Vertex AI предлагает managed-решения "из коробки", тогда как у нас observability зависит от внешнего SaaS (LangSmith). Однако self-host остаётся нашим преимуществом для регулируемых клиентов.

Сигналы: Remote MCP server, Anomaly detection

AWS Bedrock: AWS выпустил AgentCore Runtime с изоляцией агентов в отдельных microVM и AG-UI протокол для generative UI (DP). Это закрывает критерии Indep. deploy/scale и Secrets/tenant, так как каждый агент теперь живёт в своей виртуальной машине (CP). Для SL2 AI Platform это прямая угроза по GAP G3 (inter-agent observability) и G4 (cost tracking), так как Bedrock предлагает managed-решение с изоляцией и resilience patterns, тогда как у нас эти возможности требуют ручной настройки. Однако Bedrock остаётся closed SaaS, что сохраняет наше преимущество для self-host клиентов.

Сигналы: AgentCore Runtime, Resilience patterns

AutoGen: В сообществе появились open-source библиотеки для security testing (175 тестов), identity (Ed25519 для агентов) и audit trails (AlgoVoi keystone) (DP). Это усиливает критерии Multi-tenancy (identity/authority) и Secrets/tenant (токены агентов), а также Prod-ops (security testing) (CP). Для SL2 AI Platform это давление на GAP G3 (inter-agent observability) и G6 (LLM gateway), так как AutoGen предлагает стандартизированные решения для безопасности и аудита, тогда как у нас эти аспекты требуют кастомной интеграции. Однако AutoGen остаётся фреймворком, а не платформой, что сохраняет наше преимущество по managed-решениям.

Сигналы: Security testing, Identity/authority

AgentForce: Salesforce представил детерминированные conversational flows через Agent Script и continuous planning systems для финансовых сервисов (DP). Это усиливает критерии Multi-tenancy (org-scoped ACL) и Code+No-code (детерминированные flows) (CP). Для SL2 AI Platform это давление на GAP G1 (visual builder), так как AgentForce предлагает более зрелые инструменты для не-разработчиков. Однако AgentForce остаётся привязанным к Salesforce-экосистеме, что ограничивает его применение для enterprise-клиентов вне этой платформы.

Сигналы: Agent Script, Continuous planning

n8n: Добавлена возможность self-improving agent skills с cognee и опубликован AI maturity framework (DP). Это усиливает критерий Code+No-code (visual builder для skill loops) (CP). Для SL2 AI Platform это давление на GAP G1 (visual builder), так как n8n предлагает более зрелые инструменты для не-разработчиков. Однако n8n фокусируется на workflow orchestration, а не на agent runtime, что сохраняет наше преимущество по multi-tenancy и git-native dev.

Сигналы: Self-improving skills, AI maturity framework

NetBox: Вышел в GA Asset Lifecycle для интеграции procurement в систему учёта инфраструктуры (DP). Это н/п для AI-agent критериев SL2 AI Platform, так как NetBox — система учёта инфраструктуры, а не agent runtime платформа.

Сигналы: Asset Lifecycle GA

GLPI: Выпущен security release 11.0.8/10.0.26 (DP). Это н/п для AI-agent критериев SL2 AI Platform, так как GLPI — система учёта ИТ-активов.

Сигналы: 11.0.8 release

Snipe-IT: Обсуждения модернизации UI и баг с company assignment через SQL (DP). Это н/п для AI-agent критериев SL2 AI Platform, так как Snipe-IT — система учёта ИТ-активов.

Сигналы: UI modernization
