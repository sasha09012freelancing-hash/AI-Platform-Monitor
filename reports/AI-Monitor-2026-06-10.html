3. Детали по платформам

Vertex AI: На неделе Google Cloud анонсировал ускорение AI inference до 92% с помощью GKE Inference Gateway (DP), что напрямую улучшает критерий Indep. scale для Vertex AI (CP). Это означает, что managed-конкуренты начинают догонять SL2 AI Platform по эффективности масштабирования агентов, особенно в enterprise-сегменте, где latency критичен. Для нас это давление на GAP G4 (отсутствие cost tracking per agent) и G3 (нет inter-agent tracing), так как Vertex теперь предлагает более прозрачную инфраструктуру для высоконагруженных агентов. Однако Vertex остаётся closed SaaS, что сохраняет наше преимущество для self-hosted клиентов.

Сигналы: GKE Inference Gateway, Claude Fable 5 GA

AWS Bedrock: AWS вывел в GA Bedrock AgentCore Runtime с изоляцией каждого агента в отдельной microVM и Strands Agents SDK для enterprise-разработки (DP). Это закрывает у Bedrock критерии Indep. deploy, Indep. scale и Secrets/tenant (CP), так как теперь агенты работают в изолированных окружениях с собственными ресурсами и секретами. Для SL2 AI Platform это прямое давление: managed-конкурент догоняет нас по изоляции и безопасности, которые были нашими ключевыми преимуществами. Затрагиваются GAP G3 (нет inter-agent tracing), G4 (нет cost tracking) и G6 (своя мультипровайдерная обвязка вместо LiteLLM). Однако Bedrock остаётся vendor-locked SaaS, что сохраняет нашу нишу для self-hosted клиентов.

Сигналы: AgentCore Runtime, Strands Agents SDK

Azure Foundry: Microsoft анонсировал Foundry Local для edge AI и расширил observability для AI-агентов на Build 2026 (DP). Foundry Local улучшает критерий Self-host, позволяя развёртывать агенты на edge-устройствах с локальным управлением (CP). Для SL2 AI Platform это сигнал о том, что конкуренты начинают закрывать GAP в edge-сценариях, где мы пока не предлагаем готовых решений. Observability-апдейты затрагивают GAP G3 (нет inter-agent tracing) и G5 (нет self-hosted observability), так как Foundry теперь предлагает более зрелые инструменты для мониторинга агентов. Однако Foundry остаётся менее гибким в плане мультипровайдерности и git-native разработки.

Сигналы: Foundry Local, Build 2026 observability

AutoGen: В сообществе AutoGen появились предложения по криптографическим протоколам SR-LoopShield и Memory Guard для защиты от runaway loops и memory poisoning (DP). Эти инициативы затрагивают критерий Unified API + контракт (⚠) и Secrets/tenant (⚠), так как предлагают стандартизированные подходы к безопасности межагентных взаимодействий (CP). Для SL2 AI Platform это сигнал о том, что open-source конкуренты начинают закрывать GAP в области governance и безопасности, где мы пока полагаемся на собственные решения (GAP G2 — несовместимость с A2A). Однако эти предложения пока находятся на стадии обсуждения и не реализованы в коде.

Сигналы: SR-LoopShield, Memory Guard

AgentForce: Salesforce выпустил автономные AI-агенты для lead qualification и интеграцию с Claude Code для production-ready разработки (DP). Это улучшает критерии Multi-tenancy (GA org-scoped секреты) и Code+No-code (Claude Code + Skills) (CP). Для SL2 AI Platform это давление на GAP G1 (нет visual builder) и G7 (молодой проект без marketplace), так как AgentForce предлагает более зрелые инструменты для enterprise-разработчиков. Однако AgentForce остаётся привязанным к экосистеме Salesforce, что ограничивает его применение за её пределами.

Сигналы: Autonomous lead qualification, Claude Code integration

n8n: n8n внедрил OpenTelemetry для workflow tracing и опубликовал Production AI Playbook для сложных агентов (DP). Это улучшает критерий Prod-ops (↑) и LLM routing (↑), так как теперь n8n предлагает более зрелые инструменты для observability и маршрутизации (CP). Для SL2 AI Platform это сигнал о том, что конкуренты начинают закрывать GAP G3 (нет inter-agent tracing) и G4 (нет cost tracking), предлагая более прозрачные инструменты для production-разработки. Однако n8n остаётся менее гибким в плане git-native разработки и мультипровайдерности.

Сигналы: OpenTelemetry, Production AI Playbook
