3. Детали по платформам

Vertex AI: Google анонсировал 13 демо для Gemini Enterprise Agent Platform, подчёркивая интеграцию с BigQuery и IAM-метками для column-level security (DP). Это усиливает Secrets/tenant (CP) за счёт глубокой интеграции с Google Cloud IAM, что для SL2 AI Platform означает давление на GAP G9 (Secrets/tenant) — Google предлагает более зрелую экосистему для enterprise-безопасности. Однако Vertex AI остаётся managed-решением без self-host, что сохраняет наше преимущество для регулируемых клиентов. Также отмечается лидерство в Gartner MQ для Conversational AI, что может усилить привлекательность платформы для enterprise-сегмента.

Сигналы: 13 demos Gemini Enterprise, IAM Data Governance Tags

AWS Bedrock: AWS выпустил Grok 4.3 на Bedrock с поддержкой tool calling, structured output и MCP-серверов для визуальных агентов (DP). Это закрывает критерии Indep. deploy и Indep. scale за счёт AgentCore Runtime с изоляцией в microVM (CP). Для SL2 AI Platform это прямое давление на GAP G3 (observability) и G4 (FinOps) — Bedrock теперь предлагает cost tracking per agent/model "из коробки", тогда как у нас это отсутствует. Однако Bedrock остаётся closed SaaS, что сохраняет наше преимущество для self-host клиентов.

Сигналы: Grok on Bedrock, Agentic vision with MCP servers

AutoGen: В обсуждениях на GitHub подняты темы stateful-агентов, распределённого трейсинга (Otel) и AgentGraph для trust verification (DP). Это затрагивает критерий Prod-ops (CP), где AutoGen движется к паритету с SL2 AI Platform по observability (G3). Для нас это сигнал к ускорению внедрения OpenTelemetry для межагентных взаимодействий, иначе рискуем отстать в enterprise-готовности.

Сигналы: Otel tracing discussion, AgentGraph trust verification

AgentForce: Salesforce анонсировал Headless 360 MCP Server Beta для интеграции Slack-Salesforce (DP). Это усиливает Multi-tenancy и Unified API + контракт (CP) за счёт стандартизированного MCP-интерфейса. Для SL2 AI Platform это давление на GAP G2 (Agent Protocol lock-in) — Salesforce продвигает свой формат MCP, который может стать стандартом для enterprise-интеграций. Однако их решение остаётся привязанным к Salesforce-экосистеме, что ограничивает универсальность.

Сигналы: Headless 360 MCP Server Beta, Slack-Salesforce integration

n8n: Выпущены релизы с поддержкой MCP workflows и сравнение Claude Code vs n8n (DP). Это усиливает критерий Code+No-code (CP) за счёт расширения возможностей no-code оркестрации. Для SL2 AI Platform это косвенное давление на GAP G1 (visual builder) — n8n предлагает более зрелый визуальный интерфейс для не-разработчиков, хотя и не покрывает dual-mode (YAML/code) как у нас.

Сигналы: v1.123.66 release, Claude Code vs n8n

NetBox: Релиз v4.6.5 с новыми типами коннекторов и анонс NetBox Evolve 2026 (DP). Для SL2 AI Platform это н/п, так как NetBox фокусируется на инфраструктурном моделировании, а не на AI-агентах. Однако анонс конференции может усилить интерес к open-source платформам для enterprise, что косвенно выгодно нам.

Сигналы: v4.6.5 release, NetBox Evolve 2026
