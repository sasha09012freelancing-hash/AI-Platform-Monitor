3. Детали по платформам

Vertex AI: Google продолжает укреплять корпоративный сегмент, внедряя глубокие инструменты для работы с данными и обеспечения безопасности в госсекторе. Упор делается на бесшовную аналитику через BigQuery и интеграцию с Gemini Enterprise.

Сигналы: Интеграция Gemini Enterprise с A2UI, Безопасность для госсектора

AWS Bedrock: AWS сосредоточились на вопросах observability и тестирования агентов, предоставляя разработчикам глубокую аналитику от GPU до качества ответов LLM. Активно продвигается методология оценки «Deep Agents» в связке с LangSmith и Bedrock AgentCore.

Сигналы: Observability для LLM в SageMaker, Оценка Deep Agents

Azure Foundry: Платформа переходит к этапу зрелости с релизом трассировочной оценки агентов (trace-based evaluation) для любых облаков. Также расширена доступность моделей и инструментов локального исполнения Foundry Local.

Сигналы: Что нового в Microsoft Foundry май 2026

AutoGen: Сообщество активно обсуждает вопросы governance, безопасности tool-execution и аутентификации агентов в multi-agent системах. Появляются первые попытки стандартизации аудита и «доказательств завершения» (completion proofs).

Сигналы: Governance и безопасность инструментов, Аутентификация агентов

AgentForce: Salesforce развивает экосистему через поддержку Hosted MCP серверов, позволяя Claude Code напрямую работать с платформой. Это расширяет возможности управления CRM-процессами через сторонние агентские инструменты.

Сигналы: Интеграция с Claude Code, Подключение через Hosted MCP

LangGraph: Идет планомерное улучшение SDK, направленное на стабильность и производительность графовых процессов. Версия 1.2.2 задает новые стандарты совместимости в стеке LangChain.

Сигналы: Релиз LangGraph SDK 0.4.0, Обновление LangGraph 1.2.2

n8n: Платформа активно усиливает направление суверенного ИИ (sovereign AI) и контроля за выполнением агентов, предоставляя инструменты для настройки guardrails и ограничений действий. Активное внедрение поддержки MCP подчеркивает готовность к интеграции в современные агентские пайплайны.

Сигналы: Автоматизация суверенного ИИ, Надежность агентов и ограничения

5. Выводы для команды

  • Команде SL2 AI Platform следует форсировать реализацию нативных инструментов для трассировки и аудита (prod-ops), так как AWS и Azure делают это основным рыночным стандартом.
  • Необходимо рассмотреть внедрение поддержки протокола MCP, который становится де-факто стандартом для взаимодействия агентов с внешними API и CRM-системами.
  • Успех n8n с «суверенным ИИ» подтверждает спрос на self-hosted решения с продвинутыми настройками безопасности — SL2 AI Platform должна подчеркивать эти преимущества в маркетинге.
  • Важно обеспечить соответствие контрактов в Unified API стандартам открытых инструментов (типа AutoGen/LangGraph), чтобы упростить миграцию Enterprise-клиентов на SL2 AI Platform.
  • Рекомендуется провести анализ возможностей по реализации «Governance hooks» (kill-switches и audit-trails для агентов) для закрепления лидерства в Enterprise-сегменте безопасности.
