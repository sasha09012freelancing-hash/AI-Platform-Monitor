3. Детали по платформам

Vertex AI: На неделе Google опубликовал 13 hands-on демо для Gemini Enterprise Agent Platform, демонстрирующих интеграцию с BigQuery, инструменты для токен-оптимизации и расширенные возможности мультиагентных систем (DP). Это усиливает позицию Vertex AI по критерию Prod-ops, так как демо показывают зрелые практики observability и оркестрации агентов (CP). Для SL2 AI Platform это давление на GAP G3 (нет inter-agent distributed tracing) и G7 (отсутствие community и готовых интеграций), так как Google активно продвигает экосистемные решения. Однако Vertex AI остаётся managed-сервисом без self-host опций, что сохраняет наше преимущество для enterprise-клиентов с регуляторными требованиями.

Сигналы: 13 демо Gemini Enterprise Agent Platform, Gartner MQ Leader 2026

AWS Bedrock: AWS выпустил Grok 4.3 на Bedrock с поддержкой tool calling, structured output и stateful multi-turn conversations, а также усилил AgentCore Runtime изоляцией агентов в microVM (DP). Это закрывает у Bedrock критерии Indep. scale и Secrets/tenant, так как microVM обеспечивает аппаратную изоляцию и контроль доступа (CP). Для SL2 AI Platform это прямое давление на GAP G3 (нет inter-agent tracing) и G4 (нет cost tracking per agent), так как Bedrock теперь предлагает managed-решение с аналогичными enterprise-возможностями. Однако Bedrock остаётся SaaS-платформой без self-host опций, что сохраняет наше преимущество для клиентов с требованиями к data residency.

Сигналы: Grok on Amazon Bedrock, Agentic vision с MCP servers

AutoGen: В обсуждениях на GitHub появились темы по безопасности (175-тестовый фреймворк для agent security), распределённому трейсингу через OpenTelemetry и stateful-сущностям агентов (DP). Это указывает на развитие критерия Prod-ops, так как сообщество активно ищет решения для observability и безопасности мультиагентных систем (CP). Для SL2 AI Platform это затрагивает GAP G3 (нет inter-agent distributed tracing) и G6 (собственная мультипровайдерная обвязка вместо LiteLLM), так как AutoGen движется в сторону стандартизации инструментов для enterprise-разработки. Однако AutoGen остаётся open-source проектом без managed-сервиса, что сохраняет наше преимущество по self-host и multi-tenancy.

Сигналы: Open-source security testing, Improved Tool Calling Context

AgentForce: Salesforce анонсировал Headless 360 MCP Server Beta, который позволяет интегрировать Slack с Salesforce через hosted MCP servers (DP). Это усиливает критерии Multi-tenancy (изоляция данных между клиентами) и Unified API + контракт (стандартизированный MCP-интерфейс) (CP). Для SL2 AI Platform это давление на GAP G2 (Agent Protocol не совместим с A2A) и G7 (отсутствие marketplace агентов), так как Salesforce активно развивает экосистему интеграций. Однако AgentForce остаётся SaaS-решением без self-host опций, что сохраняет наше преимущество для клиентов с требованиями к data sovereignty.

Сигналы: Headless 360 MCP Server Beta, Connect Slack to Salesforce

n8n: На неделе вышли статьи по сравнению RAG vs fine-tuning и Claude Code vs n8n, а также релизы версий с поддержкой MCP workflow executions (DP). Это усиливает критерий Prod-ops, так как n8n развивает инструменты для оркестрации workflows и интеграции с AI-системами (CP). Для SL2 AI Platform это затрагивает GAP G1 (нет visual builder) и G3 (нет inter-agent tracing), так как n8n предлагает более зрелые инструменты для no-code разработки. Однако n8n не позиционируется как платформа для AI-агентов, что ограничивает прямое давление на нашу нишу.

Сигналы: Fine-Tuning vs. RAG, v1.123.66 release

NetBox: Вышел релиз v4.6.5 с новыми типами коннекторов (MDC) и обсуждения по моделированию EVPN Ethernet Segments (DP). Это не напрямую затрагивает критерии SL2 AI Platform, но показывает активность в смежной нише инфраструктурного моделирования (CP). Для нас это сигнал о необходимости отслеживать развитие инструментов для оркестрации и управления инфраструктурой, так как NetBox может стать основой для future-интеграций с AI-агентами.

Сигналы: v4.6.5 release, Modeling EVPN Ethernet Segments
