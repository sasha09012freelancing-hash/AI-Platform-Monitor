3. Детали по платформам

Vertex AI: Google выпустил Agent Development Kit (ADK) для создания масштабируемых multi-agent систем с поддержкой полиморфных агентов (DP). Это усиливает критерий Prod-ops за счёт улучшенной observability и управления динамическими бизнес-правилами (CP). Для SL2 AI Platform это давление на GAP G3 (нет inter-agent tracing) и G7 (зрелость community), так как Google предлагает enterprise-ready инструменты "из коробки". Однако ADK остаётся частью managed-сервиса, что сохраняет наше преимущество в self-host для регулируемых клиентов.

Сигналы: Beyond Static Prompts with ADK, Claude apps gateway

AWS Bedrock: AWS объявил о GA AgentCore Runtime с изоляцией каждого агента в отдельной microVM и представил serverless A2A-шлюз для маршрутизации между агентами (DP). Это закрывает критерии Indep. deploy/scale и Secrets/tenant, ранее бывшие слабыми местами Bedrock (CP). Для SL2 AI Platform это прямая угроза по GAP G3 (observability) и G4 (cost tracking), так как AWS теперь предлагает managed-решение с enterprise-grade изоляцией и прозрачностью. Однако Bedrock остаётся SaaS-платформой, что сохраняет наше преимущество в self-host.

Сигналы: AgentCore Runtime GA, Serverless A2A gateway

AutoGen: Сообщество AutoGen активно развивает open-source инструменты для security-тестирования агентов (AG-001–AG-010) и представило AgentGraph — инфраструктуру для trust-верификации агентов (DP). Это усиливает критерий Secrets/tenant за счёт поддержки Ed25519-аутентификации и проверки подлинности агентов (CP). Для SL2 AI Platform это давление на GAP G6 (LLM gateway) и G9 (secrets management), так как AutoGen предлагает готовые решения для security-сценариев, которые у нас требуют кастомной интеграции. Однако AutoGen остаётся фреймворком, а не платформой, что сохраняет наше преимущество в enterprise-ready инфраструктуре.

Сигналы: Security testing framework, AgentGraph trust verification

AgentForce: Salesforce выпустил гайд по безопасности MCP-серверов, включая аутентификацию, авторизацию и логирование, а также представил LWC State Managers для реактивного управления состоянием (DP). Это усиливает критерий Code+No-code за счёт расширения возможностей low-code разработки (CP). Для SL2 AI Platform это давление на GAP G1 (visual builder), так как Salesforce предлагает более зрелые инструменты для не-разработчиков. Однако AgentForce остаётся частью экосистемы Salesforce, что ограничивает его применение вне CRM-контура.

Сигналы: MCP Server Security Guide, LWC State Managers

n8n: Команда n8n опубликовала архитектурные гайды по error handling для LLM tool calling и безопасности MCP-серверов (DP). Это усиливает критерий Prod-ops за счёт улучшенной observability и стратегий восстановления (CP). Для SL2 AI Platform это давление на GAP G3 (inter-agent tracing) и G5 (self-hosted observability), так как n8n предлагает практические рекомендации по enterprise-grade эксплуатации. Однако n8n остаётся визуальным конструктором, что ограничивает его гибкость для разработчиков.

Сигналы: Error Handling Guide, MCP Security Guide

NetBox: NetBox Labs выпустила HIPAA-валидационный пак для проверки сетевой инфраструктуры на соответствие требованиям безопасности (DP). Это усиливает критерий Prod-ops за счёт интеграции compliance-проверок в CI/CD (CP). Для SL2 AI Platform это косвенное давление на GAP G5 (data residency), так как NetBox предлагает инструменты для offline-валидации, что актуально для регулируемых клиентов. Однако NetBox фокусируется на сетевой инфраструктуре, а не на AI-агентах.

Сигналы: HIPAA Compliance Pack

Snipe-IT: В обсуждениях появился запрос на API-аутентификацию для автоматизации работы с asset labels (DP). Это потенциально усиливает критерий Secrets/tenant, если будет реализовано (CP). Для SL2 AI Platform это сигнал о растущем спросе на автоматизацию security-операций, что может повлиять на GAP G9 (secrets management). Однако Snipe-IT — это asset management, а не AI-платформа, поэтому давление косвенное.

Сигналы: API Auth Discussion
