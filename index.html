3. Детали по платформам

Vertex AI: Платформа сфокусирована на усилении связки агентных алгоритмов с enterprise-данными из BigQuery и Google Sheets. Акцент сделан на безопасности публичного сектора и интеграции Gemini Enterprise.

Сигналы: интеграция Gemini Enterprise и A2UI, аналитика BigQuery в Google Sheets

AWS Bedrock: AWS активно развивает инфраструктуру для оценки агентов, выпуская инструменты для мониторинга качества и производительности (AgentCore). Расширяется поддержка MLflow для создания порталов управления моделями.

Сигналы: управление тестами в Bedrock AgentCore, наблюдаемость для LLM в SageMaker

Azure Foundry: Основной прорыв связан с развитием локальных возможностей Foundry для гибридных сценариев. Улучшена поддержка управления затратами и новые возможности для on-device моделей.

Сигналы: релиз Foundry Local 1.2 с поддержкой аудио и видео

AutoGen: Сообщество сфокусировано на решении проблем безопасности, аутентификации и аудита в многоагентных системах. Идет активная работа по созданию стандартов "proof of completion" для сложных рабочих процессов.

Сигналы: протоколы аутентификации агентов, governance и аудит инструментов для агентов

AgentForce: Salesforce расширяет возможности управления агентами, внедряя поддержку Claude Code и Hosted MCP Servers. Это позволяет Salesforce-разработчикам использовать внешние инструменты в рамках единого периметра платформы.

Сигналы: интеграция Claude через Salesforce MCP, разработка агентских навыков с Claude Code

LangGraph: Неделя была посвящена техническим улучшениям SDK и CLI, обеспечивающим стабильность работы графов. Улучшена совместимость с провайдерами моделей через специализированные библиотеки LangChain.

Сигналы: релиз LangGraph SDK 0.4.0, обновление CLI для управления графами

n8n: Платформа акцентирует внимание на Sovereign AI и надежности агентных исполнений. Добавлена возможность управления доступом к протоколу MCP, что приближает их к корпоративным стандартам безопасности.

Сигналы: автоматизация с Sovereign AI и Nextcloud, интеграция MCP в визуальный редактор

5. Выводы для команды

  1. Ускорить развитие собственного слоя обеспечения Governance и аудита, так как конкуренты активно закрывают этот пробел (AutoGen, n8n).
  2. Рассмотреть реализацию нативной поддержки MCP (Model Context Protocol) в SL2 AI Platform для упрощения интеграции с экосистемой сторонних инструментов.
  3. Усилить маркетинговый акцент на превосходстве SL2 AI Platform в части "indep. scale" и "indep. deploy" по сравнению с облачными Managed-решениями, которые становятся всё более закрытыми.
  4. Убедиться, что наш механизм Secrets/tenant обеспечивает изоляцию, сопоставимую с новыми enterprise-требованиями, которые задают лидеры рынка.
