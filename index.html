3. Детали по платформам

Vertex AI: Google активно продвигает концепцию «vibe coding» и внедряет инструменты для безопасного жизненного цикла AI-приложений. Ключевое внимание уделено feature flags и улучшению управления секретами в контексте безопасности API-ключей.

Сигналы: интеграция AI Studio с Cloud Run, внедрение feature flags для AI, рекомендации по безопасности API ключей

AWS Bedrock: AWS делает ставку на Bedrock AgentCore Runtime, предлагая полноценную среду для мульти-теннантных агентных систем. Фокус смещен на устранение ограничений контекстного окна через рекурсивные модели и поддержку MCP для интеграции с внешними инструментами.

Сигналы: архитектура мульти-теннантных агентов в AgentCore, преодоление барьера контекстного окна, интеграция MCP сервера в Bedrock

Azure Foundry: Платформа концентрируется на операционной эффективности моделей, предлагая инструменты для оценки качества и стоимости работы роутеров. Основная цель — обеспечить прозрачность выбора модели в production-сценариях.

Сигналы: инструменты для оценки модели-роутера

AutoGen: Сообщество сфокусировано на безопасности и доверии, внедряя стандарты проверки поведения агентов и инструментов через протоколы типа MCP Observatory. Разрабатываются механизмы верификации полномочий для автономных цепочек агентов.

Сигналы: верификация доверия к MCP инструментам, поведенческий скоринг инструментов, безопасная проверка учетных данных для кода

AgentForce: Salesforce трансформирует свою экосистему в сторону Headless 360, превращая все функции CRM в API-инструменты для агентов. Упор сделан на возможность встраивания кастомных UI-компонентов непосредственно в интерфейс агентов для enterprise-задач.

Сигналы: развитие Headless 360 и API-first подхода, использование Lightning Types в агентских сценариях

LangGraph: Продолжается итеративное улучшение стабильности рантайма, оптимизация чекпоинтов и расширение поддержки различных провайдеров моделей. Упор делается на надежность выполнения графовых структур агентов в production.

Сигналы: релиз обновления чекпоинтов 4.1.1, стабилизация SDK 0.3.15

n8n: Платформа подчеркивает разрыв между уверенностью C-level руководителей в AI и отсутствием реальных систем управления (governance). Развитие продукта идет в сторону обеспечения надежности событийных архитектур для enterprise.

Сигналы: аналитика зрелости AI в компаниях, гайд по реализации event-driven архитектуры

5. Выводы для команды

  1. AWS Bedrock активно захватывает нишу мульти-теннантности, поэтому нам важно подчеркивать преимущество SL2 AI Platform в сценариях, требующих полного self-hosted контроля (где облачные API Bedrock неприемлемы для регуляций).
  2. Стандартизация вокруг MCP (Model Context Protocol) становится промышленным стандартом — необходимо убедиться, что SL2 AI Platform обладает нативной поддержкой MCP для обеспечения интероперабельности с экосистемой AutoGen и AWS.
  3. Проблема governance и управления агентами становится критической для энтерпрайза. Интеграция фич контроля (feature flags, аудит, скоринг безопасности) в SL2 AI Platform должна стать приоритетом в roadmap.
  4. Вдохновиться подходом AgentForce по встраиванию UI-компонентов в агентские скрипты: SL2 AI Platform выиграет от предоставления аналогичных инструментов для создания «rich UI» внутри агентских workflow.
