3. Детали по платформам

Vertex AI: Google активно развивает инфраструктуру для масштабирования AI-агентов, интегрируя их с GKE Autopilot и аналитическими сервисами. Упор сделан на работу с неструктурированными данными через MCP-серверы.

Сигналы: масштабирование агентов на GKE, интеграция с GCS через MCP

AWS Bedrock: AWS сосредоточились на Enterprise-операциях, представив автоматизированный мониторинг Bedrock Ops Alert и продвинутые инструменты для оценки точности вызова инструментов (tool-calling). Платформа движется в сторону самодиагностируемых AI-систем.

Сигналы: мониторинг Bedrock Ops, улучшение tool-calling через SFT/DPO

Azure Foundry: Microsoft представила Foundry Local для Edge-разработки и Managed Compute для open-source моделей, закрывая разрыв между облаком и локальной средой. Особое внимание уделено управлению жизненным циклом агентов и надежности памяти.

Сигналы: Edge AI с Foundry Local, улучшение надежности памяти агентов

AutoGen: Сообщество активно работает над безопасностью и предотвращением бесконечных циклов в многоагентных системах. Предложены протоколы для защиты памяти от отравления и системы governance для управления действиями агентов.

Сигналы: защита памяти агентов, предотвращение бесконечных циклов

AgentForce: Salesforce расширяет экосистему за счет MCP-серверов и усиления безопасности UI-компонентов. Основной акцент делается на переносе пилотных проектов в продакшн через отработанные корпоративные паттерны.

Сигналы: MCP для Marketing Cloud, советы по переводу агентов в продакшн

LangGraph: Платформа фокусируется на стабилизации ядра и расширении поддержки различных LLM-провайдеров через плагины. Инфраструктурные изменения направлены на повышение надежности графовых вычислений.

Сигналы: релиз langgraph 1.2.4, поддержка DeepSeek

n8n: Обновления направлены на углубленное логирование, мониторинг и отладку агентных пайплайнов в продакшн-средах. Компания публикует экспертный контент по снижению галлюцинаций и управлению метриками агентов.

Сигналы: метрики производительности агентов, отладка поведения агентов

5. Выводы для команды

  1. Стандартизация через MCP (Model Context Protocol) становится новым де-факто рыночным требованием; необходимо обеспечить 100% поддержку MCP в SL2 AI Platform.
  2. Конкуренты смещают фокус с «создания агентов» на «операционную надежность»: мониторинг галлюцинаций, контроль доступа к памяти и предотвращение циклов — ключевые фичи для Enterprise-клиентов.
  3. Azure Foundry агрессивно атакует нишу Self-hosted через Foundry Local — нужно подчеркивать преимущество SL2 AI Platform как независимого от конкретного провайдера (cloud-agnostic) решения.
  4. Проактивный мониторинг безопасности (как в AutoGen) — зона роста. SL2 AI Platform должна внедрить аналогичные "guardrails" для защиты tenant-секретов и памяти.
